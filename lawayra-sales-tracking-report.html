// Example tagged URL: https://ayahuascaincolombia.com/?utm_source=instagram&utm_medium=social&utm_campaign=link-in-bio

GTM + GA4 Conversion Tracking

Five GTM tags that track the full funnel: website visit → checkout start → booking complete. Plus UTM persistence across sessions.

5

3
GA4 Events

GTM Variables
30d
UTM Cookie Life
Tag 1: UTM Parameter Capture & Storage
Custom HTML

Purpose: Captures UTM parameters from the URL on arrival, stores in a 30-day cookie and localStorage. Persists through the entire booking flow.

Trigger: All Pages

What it does: Reads utm_source, utm_medium, utm_campaign, utm_content, and utm_term from the URL. Stores as JSON in lw_utm cookie with 30-day expiry.

Tag 2: GA4 — Begin Checkout
GA4 Event

Purpose: Fires when someone clicks “Book Now” and heads to BookingLayer.

Event Name: begin_checkout

Trigger: Click — links containing your BookingLayer domain

Parameters: utm_source, utm_medium, utm_campaign (from cookie)

Tag 3: GA4 — Purchase Event
GA4 Event

Purpose: Fires on BookingLayer’s confirmation page when a booking is completed.

Event Name: purchase

Trigger: BookingLayer’s built-in dataLayer push (requires GTM integration enabled in BookingLayer Settings → Integrations)

Parameters: transaction_id, value, currency, items

Tag 4: Google Ads Conversion
Conversion Tag

Purpose: Tells Google Ads when a booking happens so it can optimize ad delivery.

Trigger: Same as Tag 3 — purchase event

Requires: Conversion ID and Label from Google Ads → Tools → Conversions

Tag 5: Meta Pixel Events
Custom HTML

InitiateCheckout: Fires on Book Now click (same trigger as Tag 2)

Purchase: Fires on booking confirmation (same trigger as Tag 3)

Includes: Transaction value and currency for ROAS calculation

Tab 1: Raw Bookings
Auto-populated

One row per booking. Zapier writes columns A, D-I, O. Columns B, C, J are auto-calculated formulas. Columns K-N are manual or cookie-based.

Tab 2: Monthly Dashboard
Summary View

Rows: All 20 source options + TOTAL row

Columns: Source | Bookings per month | Revenue per month (repeating for each month)

Key formula: =COUNTIFS('Raw Bookings'!$B:$B,"2026-03",'Raw Bookings'!$I:$I,A3)

Revenue: =SUMIFS('Raw Bookings'!$H:$H,'Raw Bookings'!$B:$B,"2026-03",'Raw Bookings'!$I:$I,A3)

Tab 3: Daily Tracker
Granular View

Rows: Dates

Columns: Date | Total Bookings | Total Revenue | one column per source

Formula: =COUNTIFS('Raw Bookings'!$A:$A,A2,'Raw Bookings'!$I:$I,D$1)

Tab 4: UTM Master List
Reference

All UTM-tagged URLs from Section 1, pre-built and ready to copy-paste. Team members grab URLs from here instead of building them manually.

Tab 5: Source Mapping
Hidden Helper

Two-column lookup table: Source name → Category. Powers the VLOOKUP in Tab 1 that auto-categorizes every booking.

Section 08

Operating Procedures

Daily, weekly, monthly, and quarterly routines to keep the tracking system accurate and actionable.

Daily Check
5 minutes
  • Glance at Daily Tracker tab — any bookings today? From which source?
  • If Zapier failed (missing rows), manually log from BookingLayer admin
Weekly Review
15 minutes
  • Verify Zapier pipeline is running (check Zap history for errors)
  • Fill in any missing “Self-Reported Source” data from BookingLayer admin
  • Check GA4 Realtime report — are UTMs tracking correctly?
  • Review bookings with blank attribution and investigate
Monthly Report
45 minutes
  • Run dual attribution reconciliation (Section 7)
  • Update Monthly Dashboard tab if formulas need extending
  • Calculate ROI per channel: Revenue ÷ Ad Spend = ROAS
  • Screenshot Looker Studio dashboard for team discussion
  • Review “Other” responses — should any become a new option?
Quarterly Audit
1 hour
  • Full channel review: Which sources are growing? Declining?
  • Reallocate ad budget based on ROAS by channel
  • Update UTM links if campaigns or URLs changed
  • Update BookingLayer dropdown if new sources emerged
  • Test full funnel: UTM link → website → test booking → verify tracking
